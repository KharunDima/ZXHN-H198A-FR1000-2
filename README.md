# OpenWrt для ZXHN H198A / ZTE E8820S (SPI NOR и NAND)

## Состав репозитория

| Папка / Файл | Назначение |
|---|---|
| `patch/` | Патчи для OpenWrt (DTS, конфигурация сборки, LED, скрипт апгрейда) |
| `bootloader/` | Готовые загрузчики для NAND и NOR (breed, U-Boot от DragonBluep и yuzhii0718) |
| `boot_mode_resistors_nand_nor/` | Схемы перестановки резисторов для выбора режима NAND или NOR |
| `eeprom/` | Два варианта калибровок (EEPROM / ART) |
| `stok_dump/` | Ссылка на стоковый дамп NAND чипа |
| `bard/` | Информация о чипах памяти и стоковый лог загрузки (UART) |

## Как собрать прошивку

1. **Клонируйте OpenWrt:**
   ```bash
   git clone --branch v25.12.5 --depth 1 https://git.openwrt.org/openwrt/openwrt.git 25.12.5
   cd 25.12.5
   ./scripts/feeds update -a && ./scripts/feeds install -a
   wget https://downloads.openwrt.org/releases/25.12.5/targets/ramips/mt7621/config.buildinfo -O .config
   ```
2. **Скопируйте файлы патчей:**

```bash
cp patch/mt7621_zte_e8820s-spi.dts      target/linux/ramips/dts/
cp patch/mt7621_zte_e8820s-nand.dts     target/linux/ramips/dts/
cp patch/mt7621_zte_e8820s-nand256.dts  target/linux/ramips/dts/
cp patch/mt7621.mk                      target/linux/ramips/image/
cp patch/leds                           target/linux/ramips/mt7621/base-files/etc/board.d/
cp patch/upgrade.sh                     target/linux/ramips/mt7621/base-files/lib/upgrade/platform.sh
cp patch/99-default-settings            package/base-files/files/etc/uci-defaults/
```
3. **Настройте сборку:**

```bash
make menuconfig
```
Выберите:

- Target System: MediaTek Ralink MIPS

- Subtarget: MT7621 based boards

- Target Profile: ZTE E8820S-SPI

LuCI: LuCI → Collections → luci (или luci-ssl)

Сохраните и выйдите.

4. **Соберите прошивку:**

```bash
make -j$(nproc) defconfig download clean world
```
Готовые образы появятся в bin/targets/ramips/mt7621/.

## Прошивка загрузчика (обязательный шаг для NAND)

**Важно!** Прошивка загрузчика — операция высокого риска. Ошибка может превратить устройство в «кирпич». Внимательно прочитайте инструкцию целиком перед началом. Автор не несёт ответственности за повреждённое оборудование.

### Почему стоковый загрузчик не подходит для OpenWrt?

Команды стокового U-Boot (nand write) часто **не умеют корректно формировать коды коррекции ошибок (ECC) и служебную область (OOB)** в том формате, который жестко требует неизменяемый аппаратный загрузчик (BootROM) процессора. Запись через стоковый U-Boot почти гарантированно приведет к несовпадению ECC, из-за чего BootROM не сможет прочитать первичный загрузчик (SPL), и устройство перестанет загружаться («окирпичится»).
Чтобы избежать этого, используется безопасный метод: устройство загружается во временную среду (например, initramfs OpenWrt), которая работает целиком в оперативной памяти (RAM). Это позволяет использовать правильные драйверы ядра Linux для точного расчета ECC, корректной работы с таблицей плохих блоков (BBT) и безопасной записи загрузчика в NAND.

### Что потребуется

- **USB‑UART адаптер** (3.3 В), подключённый к консольной колодке роутера (скорость 115200 бод).
- **TFTP‑сервер** на ПК (IP `192.168.1.2`, маска `255.255.255.0`).
- **ПК с Linux или Windows** (инструкция приведена для Linux; для Windows используйте `tftpd64`).
- **Файлы:**
  - `initramfs-kernel.bin` — временная система OpenWrt (ссылка).
  - `nand-u-boot-mt7621.bin` — наш кастомный загрузчик (ссылка).
- **Пароль стокового загрузчика:** Mt7621!!

### Шаг 1. Подготовка TFTP‑сервера

Скопируйте `initramfs-kernel.bin` и `nand-u-boot-mt7621.bin` в корневую папку TFTP‑сервера (например, `/home/user/tftp_share/`). Убедитесь, что сервер запущен и слушает порт 69 UDP.

### Шаг 2. Загрузка временной OpenWrt (initramfs)

1. Подключите ПК сетевым кабелем к **порту LAN1** роутера.
2. На ПК задайте статический IP: `192.168.1.2`, маска `255.255.255.0`.
3. Откройте терминал UART (115200 бод, 8N1).
4. Включите роутер и сразу нажмите **«1»**, затем Enter.
5. На приглашение `Please input boot password:` введите `Mt7621!!` и нажмите Enter. Повторите ввод при втором запросе.
6. В консоли U‑Boot выполните:
   ```bash
   setenv serverip 192.168.1.2
   setenv ipaddr 192.168.1.254
   saveenv
   tftpboot 0x82000000 initramfs-kernel.bin
   bootm 0x82000000
7.  Дождитесь загрузки OpenWrt (появится приглашение root@OpenWrt:~#).

---

### Шаг 3. Сохранение заводских калибровок (обязательно!)

Перед прошивкой **обязательно** сохраните раздел `kernel` (mtd3) — именно в нём стоковая прошивка хранит EEPROM для чипов MT7603E (2.4 ГГц) и MT7612E (5 ГГц). Без этих данных после замены загрузчика Wi‑Fi будет работать с пониженной мощностью (3 dBm для 5 ГГц).

#### Способ 1: Через веб-интерфейс LuCI (рекомендуется)

1. В работающей initramfs откройте браузером `http://192.168.1.1/`.
2. Перейдите в **System → Backup / Flash Firmware**.
3. В секции **Save mtdblock contents** выберите раздел **`kernel` (mtd3)** и нажмите **Save mtdblock**. Файл сохранится на ПК.

#### Способ 2: Через терминал

```bash
dd if=/dev/mtd3 of=/tmp/kernel_backup.bin bs=64k
```
Затем скопируйте файл на ПК через SCP или TFTP:

```bash
atftp -p -l /tmp/kernel_backup.bin -r kernel_backup.bin 192.168.1.2
```
#### Создание EEPROM из бэкапа
Полученный файл kernel_backup.bin загрузите на сайт https://yeezyio.github.io/. Сервис автоматически извлечёт калибровки и предложит скачать готовый eeprom.bin (размер 65,5 КБ). Этот файл содержит правильные RF‑параметры для обоих диапазонов.

### Шаг 4. Замена загрузчика
В консоли OpenWrt загрузите новый загрузчик:

```bash
atftp -g -r nand-u-boot-mt7621.bin -l /tmp/uboot.bin 192.168.1.2
```
Внимание! Следующая команда необратимо стирает стоковый загрузчик. Убедитесь, что файл загружен корректно:

```bash
hexdump -C /tmp/uboot.bin | head -5
```
В выводе должна быть строка MT7621 NAND ......

Прошейте загрузчик:

```bash
dd if=/tmp/uboot.bin of=/dev/mtdblock0 bs=2048
sync
```
Пояснение: Используется утилита dd с размером блока 2048 байт (страница NAND). Драйвер MTD автоматически рассчитывает и записывает ECC в OOB. Команда flash_erase не требуется — dd перезаписывает только первые блоки, а старый загрузчик затирается.

Проверьте запись:

```bash
nanddump -l 512 -f /tmp/check.bin /dev/mtd0 2>/dev/null
hexdump -C /tmp/check.bin | head -5
```
Первые байты должны совпадать с началом nand-u-boot-mt7621.bin.

Перезагрузите роутер:

```bash
reboot
```
### Шаг 5. Первый запуск нового загрузчика
После перезагрузки вы увидите консоль U‑Boot DragonBluep. Если экран остался чёрным, проверьте:

- надёжность подключения UART;

- не перепутан ли порядок байт (попробуйте скорость 57600);

- правильность записи (повторите шаг 4).

При успешном старте нового загрузчика вы можете загружать любые прошивки через TFTP или веб‑интерфейс (http://192.168.1.1).

### Шаг 6. Восстановление калибровок Wi‑Fi
После замены загрузчика прошейте сохранённый eeprom.bin в раздел factory:

```bash
atftp -g -r eeprom.bin -l /tmp/eeprom.bin 192.168.1.2
dd if=/tmp/eeprom.bin of=/dev/mtdblock2 bs=2048
sync
reboot
```
После перезагрузки мощность 5G восстановится до 20‑23 dBm.

### Восстановление калибровок без собственного бэкапа
Если вы не сохранили заводской раздел kernel (mtd3) или потеряли файл eeprom.bin, в репозитории проекта есть папка eeprom/. В ней лежат два готовых файла калибровок, снятых с двух одинаковых роутеров ZTE ZXHN H198A.

**Важно:** Калибровки уникальны для каждого экземпляра устройства. Чужие EEPROM могут работать как лучше, так и хуже ваших родных — это зависит от конкретной партии чипов. Тем не менее, они обеспечивают мощность 5G в диапазоне 20‑23 dBm на большинстве устройств.

#### Инструкция по восстановлению из репозитория:
Скачайте один из файлов из папки eeprom/ (например, eeprom1.bin).

Загрузите его на роутер через TFTP:

```bash
atftp -g -r eeprom1.bin -l /tmp/eeprom.bin 192.168.1.2
```
Прошейте в раздел factory:

```bash
dd if=/tmp/eeprom.bin of=/dev/mtdblock2 bs=2048
sync
reboot
```
После перезагрузки проверьте мощность 5G:

```bash
iw phy phy1 info | grep -E "^\s+\* [0-9]+\.0 MHz.*dBm" | head -5
```
Если значения в районе 20‑23 dBm — калибровки подошли. Если мощность низкая (3 dBm) — попробуйте второй файл.

**Рекомендация:** Всегда сохраняйте собственные бэкапы перед экспериментами. Чужие EEPROM — это запасной вариант, а не замена родным калибровкам.

### Восстановление после неудачной прошивки (кирпич)
Если после замены загрузчика роутер перестал подавать признаки жизни, не паникуйте. В папке stok_dump/ репозитория лежит эталонный полный дамп NAND‑чипа (138 412 032 байт) от исправного устройства. Его можно прошить программатором (например, RT809H с кроваткой TSOP48), и роутер вернётся к заводскому состоянию.

**Важно!** При записи эталонного дампа в программаторе используйте тот же режим, в котором он был считан — **Hard Copy (Ignore Bad Block)**. Это гарантирует побайтовое соответствие оригиналу и корректное восстановление заводской таблицы плохих блоков.

### Почему нельзя прошить «сырой» U‑Boot через программатор?

В отличие от NOR‑памяти, каждая страница NAND содержит служебную область (OOB), где хранятся коды коррекции ошибок (ECC) и маркер плохих блоков. BootROM процессора MT7621 проверяет ECC по строго определённому алгоритму (BCH с заданным размером сектора, layout'ом OOB и специфичным полиномом).

Современные программаторы (RT809H, T56, XGecu и др.) умеют генерировать ECC, но их встроенные пресеты часто **не совпадают** с тем, что ожидает контроллер MT7621. Кроме того, для загрузки с NAND требуется не просто `u-boot.bin`, а полный образ с корректным preloader/BL2‑заголовком и правильным размещением по адресам в чипе. Поэтому залить «сырой» файл напрямую программатором недостаточно — нужен либо готовый factory‑образ NAND, либо полный дамп с рабочего роутера той же модели.

Для NOR‑чипов такой проблемы нет: ECC отсутствует, и «сырой» дамп загрузчика прошивается напрямую через программатор в режиме **Hard Copy** без какой‑либо подготовки.

**Рекомендация:** Если вы планируете часто экспериментировать с загрузчиками и кастомными прошивками, рассмотрите аппаратную модификацию — замену NAND на SPI NOR (например, W25Q512JV, 64 МБ). Это даёт:

- Возможность восстановления через дешёвый программатор‑прищепку (CH341A и аналоги) "без выпайки".
- Простую работу с дампами — без ECC, OOB и Bad Block Management.
- Быструю отладку — заливка/чтение за секунды.

Наша прошивка полностью поддерживает конфигурацию с W25Q512JV (64 МБ) на SPI NOR.

## Загрузчики (U-Boot / Breed)

В папке `bootloader/` находятся готовые загрузчики под разные конфигурации:

| Загрузчик | Flash | Источник |
|---|---|---|
| `breed/nand-breed.bin` | NAND | Breed |

| `u-boot/yuzhii0718-uboot-mt7621-dhcpd/` | NAND и NOR | yuzhii0718 (с DHCPD) |

### Самостоятельная сборка U-Boot

Если вы хотите собрать U-Boot самостоятельно, используйте проект [DragonBluep/uboot-mt7621](https://github.com/KharunDima/uboot-mt7621). Ниже приведены параметры для GitHub Actions (вкладка **Actions → Build U-Boot → Run workflow**).

#### Для NOR flash

Заполните форму следующими значениями:

| Параметр | Значение |
|---|---|
| Flash type | `NOR` |
| Partition table | `192k(u-boot),64k(u-boot-env),64k(factory),-(firmware)` |
| Kernel offset | `0x140000` |
| Reset button GPIO | `18` |
| System LED GPIO | `16` |
| CPU frequency | `880` |
| DRAM frequency | `1200` |
| DDR param | `DDR3-128MiB` |
| Baud rate | `115200` |
| Use Old DDR Timing Parameters | ✓ (поставьте галочку) |

Также необходимо добавить поддержку чипа W25Q512JV в `spi_flash_ids.c` (ID `0xef7020`).

Нажмите **Run workflow**. Готовый `u-boot-mt7621.bin` прошейте в начало SPI NOR с помощью внешнего программатора.

#### Для NAND flash

Заполните форму следующими значениями:

| Параметр | Значение |
|---|---|
| Flash type | `NAND` (не NMBM!) |
| Partition table | `512k(u-boot),512k(u-boot-env),256k(factory),-(firmware)` |
| Kernel offset | `0x140000` |
| Reset button GPIO | `18` |
| System LED GPIO | `16` |
| CPU frequency | `880` |
| DRAM frequency | `1200` |
| DDR param | `DDR3-128MiB` |
| Baud rate | `115200` |
| Use Old DDR Timing Parameters | ✓ (поставьте галочку) |

Нажмите **Run workflow**. Готовый `u-boot-mt7621.bin` прошейте в начало NAND с помощью образа initramfs OpenWrt.
